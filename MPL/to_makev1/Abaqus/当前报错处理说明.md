# 当前报错处理说明

## 现象

出现：

```text
工况完成
没有找到 results\single\*_X.csv
没有找到 results\single\*_Y.csv
```

说明求解和提取被错误地当成同一件事。

原脚本只检查了 Abaqus 命令返回值，没有检查 CSV 是否实际生成。

## 修正版改动

1. 兼容 Abaqus Python 2/3。
2. 去掉提取命令中的 `--`。
3. 检查 ODB 是否存在且大于 1 KB。
4. 检查 `_X.csv`、`_Y.csv`、`_meta.csv` 是否真实生成。
5. 提取失败时记录 `failed`，不再显示“工况完成”。

## 重新测试前

删除旧目录中的：

```text
results
logs
```

以及测试工况目录中已有但打不开的：

```text
*.odb
*.lck
```

然后只保留一个工况：

```python
INDENTER_SIDE_LENGTHS = [2.0]
INDENTATION_DEPTHS = [0.5]
```

运行：

```bat
python run_all.py
```

若仍失败，首先打开：

```text
logs\sq2p0_d0p5_extract.log
```

把完整内容发出来。

同时检查：

```text
cases\sq2p0_d0p5\sq2p0_d0p5.sta
```

最后几行是否包含：

```text
THE ANALYSIS HAS COMPLETED SUCCESSFULLY
```
